00032 // todo() is a macro that allows you to mark sections of your code as
00033 // incomplete or in need of revision.  Use a todo when the code compiles and
00034 // runs without error in most situations, but something about it needs to be
00035 // improved.  This block will track the amount of time spent in each todo's
00036 // scope, the number of times each todo is hit in the program's execution and
00037 // other information.  When the program ends, a report is printed that helps
00038 // prioritize bug fixing and features.  In release mode, todo compiles to
00039 // (noop) by default, or it can be forcibly enabled by globally defining
00040 // a macro.
00041 //
00042 // When your program is about to end, invoke todoReport(someFunction)
00043 // to call someFunction with the todo report's text (as a const char* pointer)
00044 // as the only parameter.
00045 //
00046 // There are only 2 things you need to watch out for:
00047 //  - you should not place two todo()s on the same line, since line number
00048 //    is used to make the variables names unique.
00049 //  - timing won't work correctly if the report is printed while a todo()
00050 //    is in scope
00051 // 
00052 // Example Use:
00053 /*
00054 
00055 {
00056   todo("kwg8@cornell.edu", "replace this linear search with a table look-up");
00057   for (int i = 0; i < 1000; ++i) { todo("kwg8@cornell.edu", "loop example"); sleep(1); }
00058   sleep(1000);
00059 }
00060 // ...
00061 todoReport(printf);
00062 
00063 */
00064 // This generates a report like:
00065 //
00066 //----[  todo() report by hits  ]----------------------------------
00067 //c:\tests.cpp(188):  "loop example" - kwg8@cornell.edu (hit 1000 times - 1000 ms)
00068 //c:\tests.cpp(188):  "replace this lienar search with a table look-up" - kwg8@cornell.edu (hit 1 time - 2000 ms)
00069 //----[  todo() report by time  ]----------------------------------
00070 //c:\tests.cpp(188):  "replace this lienar search with a table look-up" - kwg8@cornell.edu (hit 1 time - 2000 ms)
00071 //c:\tests.cpp(188):  "loop example" - kwg8@cornell.edu (hit 1000 times - 1000 ms)
00072 //-----------------------------------------------------------------
00073 //
00074 //
00075 // Other Info:
00076 //  - TODO_ENABLE causes todo() and todoReport() to resolve in release mode
00077 //  - TODO_DISABLE forces those symbols to disappear in debug mode
